# 데이터는 새로운 석유다
>데이터의 진정한 가치는 그것이 정제되어 소비자에게 전달된 후의 잠재력에 있다. 또한 석유와 마찬가지로 가치 사슬의 각 단게를 통해 데이터를 전달하려면 효율적인 파이프라인이 필요하다.

# 데이터 파이프라인이란?
>데이터 파이프라인은 다양한 소스에서 새로운 가치를 얻을 수 있는 대상으로 데이터를 옮기고 변환하는 일련의 과정이다.
>이는 분석, 리포팅, 머신러닝 능력의 기초가 된다.

# 누가 파이프라인을 구축할까?
>머신러닝 모델과 데이터 과학연구, 시간에 민감한 통찰력을 제공하는 데이터에 대한 수요가 높아짐에 따라 `데이터 엔지니어링`은 분석 팀의 핵심 역할로 부상했다.
>`데이터 엔지니어`의 목적은 단순히 데이터를 데이터 웨어하우스에 로드하는 것이 아니라 데이터 과학자 및 분석가와 긴밀히 협력하여 데이터를 어떻게 처리해야 하는지 파악하고
>요구사항을 확장 가능한 프로덕션 상태로 전환하는 데이터 파이프라인을 구축하는 것이다.

# 데이터 엔지니어가 갖추어야 할 기술
- SQL과 데이터 웨어하우징
    - 데이터 엔지니어는 고성능의 SQL 작성 방법을 알고 데이터 웨어하우징 및 데이터 모델링의 기본 사항을 이해한다.
- 파이선 또는 자바
    - 파이썬이나 자바가 데이터 엔지니어링에서 우위를 점하고 있는 언어이지만, Go와 같은 언어들이 등장하고 있다.
- 분산 컴퓨팅
    - 데이터 양이 많아지고 데이터를 신속하게 처리하고자 하는 요구사항이 늘어나면서 분산 컴퓨팅 플랫폼을 사용하기 시작했다.
    - 분산 컴퓨팅은 여러 시스템의 성능을 결합하여 대량의 데이터를 효율적으로 저장, 처리 및 분석한다.
    - 하둡, 아파치 스파크 등의 프레임 워크가 있다.
- 기본 시스템 관리
    - 리눅스 커맨드 라인에 능숙해야 하며 응용 프로그램 로그 분석, 크론 작업 예약, 방화벽 및 기타 보안 설정의 문제 해결과 같은 작업을 수행할 수 있어야 한다.

# 왜 데이터 파이프라인을 구축할까?
산출물을 기반으로 하는 작업, 예측 모델 등의 뒷단에서 작동하는 데이터 파이프라인을 구축해서 대시보드나 단일 지표 등의 파생물을 제공하기 위해서.

# 어떻게 구축할까?
- 오픈소스
- 상업용
- 자체개발
등의 제품을 사용하여 구축한다.

파이프라인은 단순히 구축하는것에 머무는 것이 아니라 모니터링, 유지관리 및 확장성까지 고려해야 한다.
또한 데이터를 한 번만 제공하는 것이 아니라 파이프라인을 구축하고 이를 안정적이고 안전하게 제시간에 제공하고 처리하는 인프라를 지원해야 한다.
